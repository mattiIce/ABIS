# ABIS — Object Inventory (approximate, recovered from binaries)

> **Status:** approximate. Object **names** were recovered from the compiled
> `.pbl` libraries by scanning for printable strings in both ANSI and UTF-16LE
> (the live libraries are Unicode after the 2025 migration). PowerScript bodies
> are **not** recoverable here. Regenerate with
> `python3 tools/extract_inventory.py .` (writes
> [`docs/inventory/objects.json`](inventory/objects.json)).

## How to read these numbers

- Names are de-duplicated per object kind. Truncated "prefix-ladder" fragments
  (an artifact of scanning binaries) are collapsed to the longest form, so
  counts are a reasonable **lower-confidence estimate**, not exact.
- Per-library counts include names a library **references**, not only those it
  **defines** — PowerBuilder compiles cross-library references into each `.pbl`.
- **`menu` and `structure` counts are excluded** from the tables below as
  unreliable: the `m_` prefix conflates menu objects with their (numerous) menu
  *items*, and `m_`/`s_` are short prefixes prone to false positives. They
  remain in `objects.json` for transparency only.
- A handful of legacy mixed-case names (e.g. `W_inv_Coil`, `D_STATUS`) are
  excluded because the all-lowercase filter is what keeps binary noise out.

## Application scale (distinct names, reliable kinds)

| Object kind | Approx. count |
|---|--:|
| DataWindows (`d_*`) | ~1,345 |
| Windows (`w_*`) | ~588 |
| Classes / NVOs (`n_*`, `nvo_*`) | ~181 |
| User objects (`u_*`) | ~136 |
| Global functions (`f_*`) | ~325 |
| PFC framework objects (`pfc_*`, `pfe_*`) | ~464 |

**Takeaway:** this is a large enterprise application — on the order of a few
thousand objects across 45 libraries, dominated by ~1,300+ DataWindows (the data
access + presentation layer) and ~600 windows (the UI). Any rewrite is a
multi-quarter effort; see [`MODERNIZATION_ROADMAP.md`](MODERNIZATION_ROADMAP.md).
